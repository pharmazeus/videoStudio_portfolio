ee# Docs

## Purpose

This folder stores application-facing documentation only (features, UX behavior, architecture notes tied directly to the app).

## Scope

Use `docs/` for app-related references such as:

- feature overviews
- UI/UX behavior notes
- architecture notes specific to app modules
- integration notes relevant to runtime behavior

## Continuity files

Operational continuity/logging protocol is intentionally kept outside `docs/`:

- `specs/continuity-protocol.md`
