# ui-builder skill

An agent skill that builds UI components anchored to your project's actual design system —
not generic AI output. Every component it generates matches your tokens, your visual history,
and your conventions. Design drift stops here.

---

## What it does

- Reads your design system before writing a single line of code
- Loads per-component specs and visual references before building each component
- Enforces design tokens — no hardcoded hex, no arbitrary Tailwind values
- Saves specs after building so future sessions stay consistent
- Flags gaps in your design system instead of silently inventing

Works with React, Next.js, Vue, HTML/CSS — any stack using Tailwind or CSS custom properties.

Copy the skill folder into your project:

Upload `SKILL.md` and all reference files as project knowledge.
Place `design-system.md` in the root of your project knowledge.

## Setup (required before first use)

### Step 1 — Fill in your design system

Open `design-system.md` and replace every `[FILL IN]` and `[hex]` placeholder
with your actual project values:

```markdown
primary: #6366f1  → bg-primary / text-primary
```

This is the most important step. The skill is only as good as this file.
Takes 15–20 minutes. Do it once, maintain it as your tokens change.

**Where to find your values:**
- `tailwind.config.ts` — color and spacing tokens
- `src/styles/globals.css` — CSS custom properties (`--primary`, `--background`, etc.)
- Your Figma file — color styles, text styles, spacing grid

### Step 2 — Fill in the component specs

Each file in `references/components/` has sections marked `[FILL IN]` or `[project decision]`.
The most important ones:

| File | What to fill in |
|------|----------------|
| `button/spec.md` | Confirm border radius rule for your project |
| `card/spec.md` | Confirm shadow level and padding scale |
| `input/spec.md` | Confirm focus ring color and error state behavior |
| `nav/spec.md` | Choose sidebar vs top nav, fill in dimensions |
| `design-system.md` | Icon library, component library (shadcn/custom), forbidden patterns |

### Step 3 — Add visual references (optional but powerful)

For each component you've already built, screenshot it and save it:

```
references/components/button/reference.png    ← what your button looks like
references/components/card/reference.png      ← what your card looks like
```

The agent will match this visual in future sessions. Without images, it works from
spec alone — still consistent, but less visually precise.

## Usage

The skill triggers automatically. Just describe what you want:

```
"Build a stats card for the dashboard"
"Make a delete button with confirmation dialog"  
"Style this form — it looks off"
"Update this card to match the rest of the UI"
"Build a sidebar nav with these 5 sections"
```

It also triggers when you share a screenshot or mockup:

```
"Make it look like this" + [image]
"Match this Figma export" + [image]
```

### Explicit activation (any agent)

```
Use the ui-builder skill to build a notification badge component.
```

## Reference library — how to grow it

The skill gets better the more references you add. Two types:

### Component references
After building a new component, screenshot it and save:

```
references/components/[name]/
├── spec.md        ← generated by the skill after building
└── reference.png  ← your screenshot
```

The skill will prompt you to do this after each new component.

### Page references
When a full page is working, save it:

```
references/screens/[page-name]/
├── notes.md       ← layout rules, what matters on this page
└── reference.png  ← full page screenshot
```

### Inspiration images
When you find a UI pattern you want to borrow from another product:

1. Save the screenshot to `references/inspiration/[name].png`
2. Add an entry to `references/inspiration/index.md`:

```markdown
## dashboard-layout.png
Source: https://linear.app (screenshot)
Extract: sidebar width, nav item spacing, header height
Ignore: their colors, fonts, brand identity
```

The skill reads `index.md` before looking at any inspiration image.

## File reference

| File | Purpose | Edit frequency |
|------|---------|---------------|
| `SKILL.md` | Skill instructions (do not edit) | Never |
| `design-system.md` | Your design tokens and rules | When tokens change |
| `references/components/*/spec.md` | Per-component specs | After each new component |
| `references/components/*/reference.png` | Visual targets | After each new component |
| `references/screens/*/notes.md` | Page layout rules | When page layout solidifies |
| `references/inspiration/index.md` | Inspiration image index | When adding inspiration |
| `evals/evals.json` | Trigger accuracy tests | Rarely |

## Testing trigger accuracy

Run the eval queries in `evals/evals.json` to verify the skill activates
when it should and stays quiet when it shouldn't.

In an agent CLI environment:
```
Run the evals in .agents/skills/ui-builder/evals/evals.json and tell me 
which prompts correctly trigger the ui-builder skill.
```

A healthy skill should trigger on all 15 `should_trigger: true` queries
and stay silent on all 8 `should_trigger: false` queries.

## Troubleshooting

**Skill isn't triggering**
- Make sure `SKILL.md` is in the correct path: `.agents/skills/ui-builder/SKILL.md`
- In your agent environment, verify the skill is loaded
- Try explicit activation: "Use the ui-builder skill to..."

**Agent is inventing colors not in my design system**
- Your `design-system.md` is incomplete — fill in the color palette section
- Add a "Forbidden patterns" entry: `❌ NO raw color classes: bg-blue-500`

**Component doesn't match my existing UI**
- Add a `reference.png` screenshot of your existing component
- The skill will match it visually in future sessions

**Spec files are getting stale**
- After changing a component significantly, ask the agent to update the spec:
  "Update references/components/button/spec.md to reflect the current implementation"

## Extending the skill

### Add a new component spec

Ask your agent:
```
Create a spec.md for our [table / toast / tooltip / etc] component 
and save it to .agents/skills/ui-builder/references/components/[name]/spec.md
```

### Add a new page reference

```
Screenshot the [dashboard / settings / onboarding] page and save it with notes 
to .agents/skills/ui-builder/references/screens/[name]/
```

### Update the design system

When your Tailwind config or CSS variables change:
```
Update design-system.md — we changed our primary color from indigo to violet 
and added a new surface color token.
```
